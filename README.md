# SKU-MATCH — деплой на Timeweb Cloud App Platform

Единый Node.js/Express-сервер: раздаёт `index.html` и обрабатывает
`POST /api/find-part` в одном процессе. Работает без VPN для пользователей
из России, так как хостинг физически в РФ.

## Файлы
- `server.js` — сервер (статика + API-эндпоинт к YandexGPT)
- `package.json` — зависимости и команда запуска
- `index.html` — сам виджет (без изменений)

## Что нужно заранее
- Аккаунт на cloud.timeweb.com
- `YANDEX_API_KEY` и `YANDEX_FOLDER_ID` (те же, что уже использовали)
- Репозиторий на GitHub с этими файлами (тот же `sku-match`, что уже есть)

## Шаги

### 1. Обновите репозиторий на GitHub
Замените в репозитории `index.html` на актуальный (если ещё не сделали) и
добавьте новые файлы `server.js`, `package.json`. Удалите папку `api/` —
она была нужна только для Vercel.

```bash
cd vercel_project
git add .
git commit -m "переход на Express для Timeweb"
git push
```

### 2. Создайте приложение в Timeweb Cloud
1. Личный кабинет Timeweb Cloud → **App Platform** → **Создать**
2. Выберите **Node.js** (или Express, если предложат отдельным пунктом)
3. Подключите GitHub-аккаунт (если ещё не подключён) → выберите репозиторий `sku-match`
4. Build-команда: `npm install`
5. Команда запуска: `npm start`
6. Порт: приложение слушает `process.env.PORT` автоматически — Timeweb сам
   подставит нужный порт, ничего вручную указывать не нужно

### 3. Переменные окружения
В настройках приложения → **Переменные окружения** добавьте:
- `YANDEX_API_KEY` = ваш ключ
- `YANDEX_FOLDER_ID` = `b1gpu4tduek1akl917l1` (или актуальный)

### 4. Запустите деплой
Нажмите **Развернуть/Deploy**. Timeweb соберёт и запустит приложение,
покажет прогресс сборки в реальном времени. По завершении вы получите
рабочий адрес вида `ваше-приложение.twc1.net`.

### 5. Проверка
Откройте полученный адрес, введите тестовый запрос ("Karcher K3, крутится
но не дует"). Проверьте с мобильного интернета без VPN — это и есть
главная цель переезда.

### 6. Подключите домен (опционально)
App Platform → ваше приложение → **Домены** → добавьте свой домен
(например, `sku-match-3d.ru`, если купите такой). Обновите QR на закладке
под новый адрес после этого шага.

## После переезда
- Ссылку из Vercel можно отключить/удалить
- QR на закладке нужно перегенерировать под новый домен — пришлите финальный
  адрес, и это будет сделано отдельным шагом
